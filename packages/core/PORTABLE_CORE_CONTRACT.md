# Portable Core Contract

Version: 1.0.0
Status: ACTIVE

## Purpose

This document defines the subset of `@codluv/llmtxt` functions that MUST produce
byte-identical output across all implementations (TypeScript npm package, Rust crate,
future WASM builds). Any consumer that depends on cross-platform interoperability
(e.g. SignalDock verifying signatures generated by a TypeScript service) relies on
this contract.

## Portable Functions

### 1. compress / decompress

- **Algorithm**: zlib deflate (RFC 1951), default compression level
- **Encoding**: Input is UTF-8 string, output is raw deflate bytes (no gzip/zlib header)
- **Contract**: `decompress(compress(input)) === input` for any valid UTF-8 string

#### Test Vectors

```
Input:  "Hello, world!"
Output: (deflate bytes, hex) â€” implementation-specific but decompress must round-trip

Input:  "" (empty string)
Output: (valid deflate of empty, must decompress to "")

Input:  "{"key":"value","nested":{"a":1}}"
Output: (deflate bytes, must decompress to exact input)
```

Note: Exact compressed bytes may vary by compression level/implementation. The
contract is round-trip correctness: `decompress(compress(x)) === x` for all inputs.
Cross-implementation decompression is the critical guarantee: bytes compressed by
TypeScript must decompress correctly in Rust and vice versa.

### 2. generateId

- **Algorithm**: `crypto.randomUUID()` â†’ remove dashes â†’ take first 16 hex chars â†’ parse as integer â†’ base62 encode â†’ pad to 8 chars (left-pad with '0', truncate to 8)
- **Base62 alphabet**: `0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz`
- **Output**: Always exactly 8 characters
- **Contract**: Output matches regex `^[0-9A-Za-z]{8}$`

Note: Output is random, so no fixed test vectors. Verify format and base62 round-trip:
`decodeBase62(encodeBase62(n)) === n` for all non-negative integers.

### 3. hashContent

- **Algorithm**: SHA-256
- **Input encoding**: UTF-8
- **Output**: Lowercase hex-encoded digest (64 characters)

### 4. calculateTokens

- **Algorithm**: `ceil(length_in_chars / 4)`
- **Input**: UTF-8 string, count Unicode code points (JavaScript `.length`)
- **Output**: Non-negative integer

#### Test Vectors

```
Input:  "Hello, world!"  (13 chars)
Output: 4

Input:  ""  (0 chars)
Output: 0

Input:  "a"  (1 char)
Output: 1

Input:  "1234"  (4 chars)
Output: 1

Input:  "12345"  (5 chars)
Output: 2
```

### 5. encodeBase62 / decodeBase62

- **Alphabet**: `0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz`
- **Encoding**: Big-endian (most significant digit first)
- **Zero**: `encodeBase62(0) === "0"`

#### Test Vectors

```
encodeBase62(0)    === "0"
encodeBase62(1)    === "1"
encodeBase62(61)   === "z"
encodeBase62(62)   === "10"
encodeBase62(3844) === "100"

decodeBase62("0")   === 0
decodeBase62("z")   === 61
decodeBase62("10")  === 62
decodeBase62("100") === 3844
```

### 6. computeSignature

- **Algorithm**: HMAC-SHA256
- **Payload format**: `"${slug}:${agentId}:${conversationId}:${expiresAt}"`
- **Output**: First 16 characters of the hex-encoded HMAC digest (lowercase)

#### Test Vectors

```
Params: { slug: "xK9mP2nQ", agentId: "test-agent", conversationId: "conv_123", expiresAt: 1700000000000 }
Secret: "test-secret"
Payload: "xK9mP2nQ:test-agent:conv_123:1700000000000"
Full HMAC-SHA256 (hex): (compute with above payload and secret)
Output: first 16 chars of that hex digest
```

### 7. verifySignedUrl

- **Input**: URL with query params `agent`, `conv`, `exp`, `sig`
- **Process**:
  1. Extract slug from pathname (strip leading `/`)
  2. Extract `agent`, `conv`, `exp`, `sig` from query string
  3. Check all 4 params present â†’ `missing_params` if not
  4. Parse `exp` as integer, compare to current time â†’ `expired` if past
  5. Reconstruct signature via `computeSignature` and compare â†’ `invalid_signature` if mismatch
- **Comparison**: Timing-safe (constant-time) comparison of signature strings

### 8. calculateCompressionRatio

- **Algorithm**: `round(originalSize / compressedSize, 2)` â€” returns 1 when compressedSize is 0

#### Test Vectors

```
calculateCompressionRatio(1000, 400)  === 2.5
calculateCompressionRatio(100, 100)   === 1.0
calculateCompressionRatio(100, 0)     === 1.0
calculateCompressionRatio(500, 200)   === 2.5
```

### 9. deriveSigningKey (Phase 4)

- **Algorithm**: `HMAC-SHA256(apiKey, "llmtxt-signing")`
- **Output**: Full hex-encoded HMAC digest (64 characters)
- **Purpose**: Derive a per-agent signing key from their API key, avoiding shared secrets

#### Test Vectors

```
Input:  apiKey = "sk_live_abc123"
Output: HMAC-SHA256("sk_live_abc123", "llmtxt-signing") as hex
```

## Non-Portable Functions

The following are TypeScript-only and NOT required in the Rust crate:

- **Validation**: `validateContent`, `detectFormat`, `autoValidate` (Rust uses serde)
- **Schemas**: All Zod schemas (Rust uses serde + validator)
- **Progressive Disclosure**: `generateOverview`, `getSection`, `searchContent`, `queryJsonPath`, `getLineRange` (text parsing, stays in content service)
- **Cache**: `LRUCache` class (Rust has its own caching solutions)

## Compatibility Testing

Both implementations MUST pass the same test vector suite. The TypeScript package
includes a `test-vectors.json` file that the Rust crate imports for validation.

Format:
```json
{
  "hashContent": [

  "computeSignature": [

  ]

```
