Type contracts exported by this package: interfaces, type aliases, and enums.

## CacheStats

Snapshot of cache performance statistics.

| Property  | Type     | Required | Description                                                      |
| --------- | -------- | -------- | ---------------------------------------------------------------- |
| `hits`    | `number` | Yes      | Total number of cache hits since last reset.                     |
| `misses`  | `number` | Yes      | Total number of cache misses since last reset.                   |
| `size`    | `number` | Yes      | Current number of live (non-expired) entries in the cache.       |
| `maxSize` | `number` | Yes      | Maximum number of entries the cache can hold.                    |
| `hitRate` | `number` | Yes      | Hit rate as a percentage (0-100), rounded to two decimal places. |

## LRUCacheOptions

Configuration options for constructing an `LRUCache` instance.

| Property  | Type                  | Required | Description                                                                                |
| --------- | --------------------- | -------- | ------------------------------------------------------------------------------------------ |
| `maxSize` | `number \| undefined` | No       | Maximum number of entries before the least-recently-used entry is evicted (default: 1000). |
| `ttl`     | `number \| undefined` | No       | Default time-to-live in milliseconds for new entries (default: 24 hours).                  |

## Section

A logical section identified within a document.

| Property     | Type                                                               | Required | Description                                                            |
| ------------ | ------------------------------------------------------------------ | -------- | ---------------------------------------------------------------------- |
| `title`      | `string`                                                           | Yes      | Display title of the section (heading text, JSON key, or symbol name). |
| `depth`      | `number`                                                           | Yes      | Nesting depth (0-based). Headings use depth = level - 1.               |
| `startLine`  | `number`                                                           | Yes      | 1-based line number where the section begins.                          |
| `endLine`    | `number`                                                           | Yes      | 1-based line number where the section ends (inclusive).                |
| `tokenCount` | `number`                                                           | Yes      | Estimated token count for the section content.                         |
| `type`       | `"function" \| "heading" \| "json-key" \| "code-block" \| "class"` | Yes      | The structural type of the section.                                    |

## DocumentOverview

High-level structural overview of a document.

| Property     | Type                                                             | Required | Description                                                           |
| ------------ | ---------------------------------------------------------------- | -------- | --------------------------------------------------------------------- |
| `format`     | `"json" \| "markdown" \| "code" \| "text"`                       | Yes      | The detected document format.                                         |
| `lineCount`  | `number`                                                         | Yes      | Total number of lines in the document.                                |
| `tokenCount` | `number`                                                         | Yes      | Estimated total token count for the entire document.                  |
| `sections`   | `Section[]`                                                      | Yes      | Ordered list of sections found in the document.                       |
| `keys`       | `{ key: string; type: string; preview: string; }[] \| undefined` | No       | Top-level JSON keys with type info and preview (JSON documents only). |
| `toc`        | `{ title: string; depth: number; line: number; }[] \| undefined` | No       | Markdown table of contents entries (markdown documents only).         |

## SearchResult

A single match returned by `searchContent`.

| Property        | Type       | Required | Description                                                   |
| --------------- | ---------- | -------- | ------------------------------------------------------------- |
| `line`          | `number`   | Yes      | 1-based line number of the matching line.                     |
| `content`       | `string`   | Yes      | The full text of the matching line.                           |
| `contextBefore` | `string[]` | Yes      | Lines immediately preceding the match (up to `contextLines`). |
| `contextAfter`  | `string[]` | Yes      | Lines immediately following the match (up to `contextLines`). |

## LineRangeResult

Result of extracting a line range from a document via `getLineRange`.

| Property      | Type     | Required | Description                                                     |
| ------------- | -------- | -------- | --------------------------------------------------------------- |
| `startLine`   | `number` | Yes      | 1-based line number where the extracted range begins.           |
| `endLine`     | `number` | Yes      | 1-based line number where the extracted range ends (inclusive). |
| `content`     | `string` | Yes      | The extracted text content for the requested line range.        |
| `tokenCount`  | `number` | Yes      | Estimated token count for the extracted content.                |
| `totalLines`  | `number` | Yes      | Total number of lines in the full document.                     |
| `totalTokens` | `number` | Yes      | Estimated total token count for the full document.              |
| `tokensSaved` | `number` | Yes      | Number of tokens saved by extracting only this range.           |

## ValidationResult

Outcome of a content validation operation.

| Property  | Type                                          | Required | Description                                                        |
| --------- | --------------------------------------------- | -------- | ------------------------------------------------------------------ |
| `success` | `boolean`                                     | Yes      | Whether the validation passed without errors.                      |
| `data`    | `T \| undefined`                              | No       | The parsed/validated data, present only when `success` is `true`.  |
| `errors`  | `ValidationError[] \| undefined`              | No       | List of validation errors, present only when `success` is `false`. |
| `format`  | `"json" \| "markdown" \| "text" \| undefined` | No       | The detected or requested content format, when determinable.       |

## ValidationError

A single validation error with location, message, and error code.

| Property  | Type     | Required | Description                                                                     |
| --------- | -------- | -------- | ------------------------------------------------------------------------------- |
| `path`    | `string` | Yes      | Dot-delimited path to the field that failed validation (empty string for root). |
| `message` | `string` | Yes      | Human-readable description of the validation failure.                           |
| `code`    | `string` | Yes      | Machine-readable error code (e.g. `"invalid_json"`, `"unknown_schema"`).        |

## SignedUrlParams

Parameters that uniquely identify a signed URL access grant.

| Property         | Type     | Required | Description                                                   |
| ---------------- | -------- | -------- | ------------------------------------------------------------- |
| `slug`           | `string` | Yes      | Short base62-encoded document identifier (e.g. `"xK9mP2nQ"`). |
| `agentId`        | `string` | Yes      | Unique identifier of the agent requesting access.             |
| `conversationId` | `string` | Yes      | Conversation scope that this access grant is bound to.        |
| `expiresAt`      | `number` | Yes      | Absolute expiration time as a Unix timestamp in milliseconds. |

## SignedUrlConfig

Configuration for generating and verifying signed URLs.

| Property  | Type     | Required | Description                                                |
| --------- | -------- | -------- | ---------------------------------------------------------- |
| `secret`  | `string` | Yes      | Shared HMAC-SHA256 secret used to sign and verify URLs.    |
| `baseUrl` | `string` | Yes      | Base URL for document access (e.g. `"https://llmtxt.my"`). |

## VerifyResult

Outcome of verifying a signed URL via `verifySignedUrl`.

| Property | Type                                                                | Required | Description                                                            |
| -------- | ------------------------------------------------------------------- | -------- | ---------------------------------------------------------------------- |
| `valid`  | `boolean`                                                           | Yes      | Whether the signature is valid and the URL has not expired.            |
| `reason` | `"missing_params" \| "expired" \| "invalid_signature" \| undefined` | No       | Machine-readable failure reason, present only when `valid` is `false`. |
| `params` | `SignedUrlParams \| undefined`                                      | No       | Reconstructed request parameters, present only when `valid` is `true`. |
