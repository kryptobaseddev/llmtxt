# OpenAPI 3.1 Specification — DX-1 (T169)

## Overview

The LLMtxt API publishes a live, auto-generated OpenAPI 3.1 specification that
stays in sync with the running backend. The spec is the single source of truth
for API consumers, Postman collections, and TypeScript client generation.

## Endpoints

| URL | Description |
|-----|-------------|
| `GET https://api.llmtxt.my/openapi.json` | Raw OpenAPI 3.1 spec (JSON) |
| `GET https://api.llmtxt.my/docs/api` | Interactive Swagger UI |

On the Fastify server, these map to `/api/openapi.json` and `/api/docs/api`
(the `api.llmtxt.my` host rewrite prepends `/api` to every request).

## Spec Quality

The spec (as of v2026.4.6) contains:

- **76 paths / 93 operations** across all API domains
- **OpenAPI version**: 3.1.0
- **Security schemes**: `ApiKeyAuth` (Bearer token), `SessionCookie`, `AgentSignature`
- **0 validation errors** against openapi-schema-validator
- Every operation has: `summary`, `operationId`, response schema(s) for 200 and most-likely errors
- POST/PUT/PATCH operations include `requestBody` schemas where applicable

## Generation

The spec is generated by `forge-ts` from TSDoc `@route` annotations in
`apps/backend/src/openapi-manifest.ts`, then post-processed by
`scripts/postprocess-openapi.mjs` which:

1. Upgrades version to OpenAPI 3.1.0
2. Fixes path parameters (`slug` → `{slug}`)
3. Injects response schemas for every operation
4. Adds security schemes (`ApiKeyAuth`, `SessionCookie`, `AgentSignature`)
5. Adds domain tags and a tags definition block
6. Merges additional routes not captured by forge-ts (blobs, export, collections, retrieval, auth)
7. Injects `servers`, `info`, and `contact` blocks

To regenerate after adding routes to the manifest:

```bash
pnpm --filter backend run openapi:gen
```

## CI Integration

The `openapi-drift` job in `.github/workflows/ci.yml` runs on every push and PR:

1. **Regenerates** the spec via `openapi:gen`
2. **Fails** if `openapi.json` or the docs-site copy is out of sync with HEAD
3. **Validates** the spec with `openapi-schema-validator` (0 errors required)
4. **Verifies** that `openapi-typescript` can generate a TypeScript client with 0 errors
5. **Uploads** `postman-collection.json` as a build artifact (30-day retention)
6. **Uploads** `openapi.json` as a build artifact (30-day retention)

## Postman Collection

A Postman collection is generated from the spec automatically and committed to:
`apps/backend/postman-collection.json`

It is also uploaded as a GitHub Actions artifact (`postman-collection`) on every
CI run. Import it directly into Postman via the artifact download URL.

## TypeScript Client Generation

The spec can generate a full TypeScript client via `openapi-typescript`:

```bash
pnpm --filter backend run openapi:ts-client
# Output: /tmp/llmtxt-api-types.ts
```

For a persistent client, run:

```bash
npx openapi-typescript https://api.llmtxt.my/openapi.json -o src/api-types.ts
```

Then use `openapi-fetch` or similar to make type-safe API calls.

## Authentication

The spec documents three auth methods:

### ApiKeyAuth (Bearer token)

```http
Authorization: Bearer llmtxt_<43-char-base64url-key>
```

Create keys via `POST /api/keys`. Keys are prefixed `llmtxt_`.

### SessionCookie

```http
Cookie: llmtxt_session=<session-token>
```

Set by the auth provider (Better Auth) after OAuth sign-in.

### AgentSignature (Ed25519)

```http
X-Agent-Pubkey-Id: <fingerprint>
X-Agent-Signature: <base64-sig>
X-Agent-Nonce: <nonce>
X-Agent-Timestamp: <unix-ms>
```

Register agent keys via `POST /api/agents/keys`.

## Adding Routes to the Spec

When adding a new route:

1. Add a TSDoc-annotated stub function to `src/openapi-manifest.ts` following the
   existing `@route`, `@body`, `@response` convention.
2. Add the `operationId` → responses entry to `RESPONSES` in `scripts/postprocess-openapi.mjs`.
3. Add the `operationId` → tag entry to `TAG_MAP`.
4. Run `pnpm --filter backend run openapi:gen` to regenerate.
5. Run `pnpm --filter backend run openapi:validate` to verify 0 errors.
6. Commit `openapi.json` and `apps/docs/public/api/openapi.json` together.
