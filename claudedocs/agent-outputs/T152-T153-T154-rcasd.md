## Architecture Decisions

### D-W3-1: BFT Threshold Formula

**Decision**: BFT quorum = 2f+1 where f = fault_tolerance (INTEGER, default 1) stored per
document. Quorum is recomputed dynamically after each agent revocation — if 2 Byzantine agents
are slashed, the active signer pool shrinks and quorum recalculates against remaining active
signers.

**Rationale**: Simple majority (n/2+1) fails at f=1 Byzantine; 2f+1 is the minimum for
Byzantine fault tolerance in a vote-based protocol. f is configurable because different
documents have different risk profiles (public wiki vs. legal contract).

**Implementation home**: `crates/llmtxt-core/src/bft.rs` (SSoT rule D001)

**Impact**: T248 (Rust impl) → T251 (schema) → T253 (conflict detector) → T259 (endpoint) → T262 (manifest)

---

### D-W3-2: Scratchpad is Redis-only — no Postgres persistence

**Decision**: Scratchpad messages live exclusively in Redis Streams (`scratchpad:{slug}`).
No Postgres table. 24h TTL enforced via XTRIM MAXLEN 10000 + hourly cleanup job scanning
`scratchpad:*` keys.

**Rationale**: Scratchpad is intentionally ephemeral coordination data, not document content.
Routing through Postgres would add write amplification, complicate migrations, and blur the
separation between durable state (Postgres) and volatile coordination (Redis). If Redis is
unavailable, scratchpad routes return 503 SERVICE_UNAVAILABLE — it is opt-in infrastructure.

**Impact**: T274 (schema/ADR) → T275 (Redis client) → T276/T280/T281/T283/T286/T287

---

### D-W3-3: A2A Envelope schema lives in crates/llmtxt-core

**Decision**: The A2AMessage struct, canonical_bytes() derivation, sign(), and verify()
live in `crates/llmtxt-core/src/a2a.rs`. The A2A envelope is a superset of the signed
approval canonical payload (same alphabetically-sorted JSON pattern from T147).

**Rationale**: SSoT rule D001 — all portable cryptographic primitives must live in the
Rust core. The TS SDK wraps WASM exports, not node:crypto. This also ensures the
canonical_bytes format is byte-identical across all language implementations.

**Impact**: T290 (spec) → T291 (Rust) → T292 (SDK) → T296/T297/T298/T300/T303
